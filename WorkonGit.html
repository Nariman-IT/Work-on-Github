Введение

Основы Git


Git — это распределительная система контроля версий, позволяющая сохранять все изменения которые происходят в файле или файлах, и дает возможность вернуться позже к определённой версии.
Git хранит данные как набор снимков. Каждый раз, когда мы делаем коммит, то есть сохраняем состояние своего проекта в Git, система запоминает, как выглядит каждый файл в этот момент, и сохраняет ссылку на этот снимок. Для увеличения эффективности, если файлы не были изменены, Git не запоминает эти файлы вновь, а только создаёт ссылку на предыдущую версию идентичного файла, который уже сохранён. Git представляет свои данные как, скажем, поток снимков.
В Git для всего вычисляется хеш-сумма, и только потом происходит сохранение. В дальнейшем обращение к сохранённым объектам происходит по этой хеш-сумме. Это значит, что невозможно изменить содержимое файла или каталога так, чтобы Git не узнал об этом. Данная функциональность встроена в Git на низком уровне и является неотъемлемой частью его философии. Вы не потеряете информацию во время её передачи и не получите повреждённый файл без ведома Git.
Механизм, которым пользуется Git при вычислении хеш-сумм, называется SHA-1 хеш. Это строка из 40 символов, состоящая из шестнадцатеричных символов (0–9 и a–f), она вычисляется на основе содержимого файла или структуры каталога.

Состояния файлов


У Git есть три основных состояния, в которых могут находиться ваши файлы: изменён (modified), индексирован (staged) и зафиксирован (committed).
К изменённым относятся файлы, которые поменялись, но ещё не были зафиксированы.
Индексированный — это изменённый файл в его текущей версии, отмеченный для включения в следующий коммит.
Зафиксированный значит, что файл уже сохранён в вашей локальной базе.

Три области Git


Рабочая директория (Working Directory). Файлы извлекаются из сжатой базы данных репозитория и помещаются на локальном диске для использования или изменения.
Область индексирования (Staging Area). Это файл, обычно содержащийся в вашем каталоге Git, в котором хранится информация о том, что войдет в ваш следующий коммит. Её техническое название на языке Git — «индекс», но фраза «область индексирования» также работает.
Каталог Git (Git directory, Repository). Это то место, где Git хранит метаданные и базу объектов вашего проекта. Это самая важная часть Git и это та часть, которая копируется при клонировании репозитория с другого компьютера.
Git config
В состав Git входит утилита git config, которая позволяет просматривать и настраивать параметры, контролирующие все аспекты работы Git, а также его внешний вид. Эти параметры могут быть сохранены в трёх местах:
I. Файл [path]/etc/gitconfig содержит значения, общие для всех пользователей системы и для всех их репозиториев. Если при запуске git config указать параметр --system, то параметры будут читаться и сохраняться именно в этот файл.
II. Файл ~/.gitconfig или ~/.config/git/config хранит настройки конкретного пользователя. Этот файл используется при указании параметра --global и применяется ко всем репозиториям, с которыми вы работаете в текущей системе.
III. Файл config в каталоге Git (т. е. .git/config) репозитория, который вы используете в данный момент, хранит настройки конкретного репозитория. Вы можете заставить Git читать и писать в этот файл с помощью параметра --local, но на самом деле это значение по умолчанию. Неудивительно, что вам нужно находиться где-то в репозитории Git, чтобы эта опция работала правильно.
Настройки на каждом следующем уровне подменяют настройки из предыдущих уровней, то есть значения в .git/config перекрывают соответствующие значения в [path]/etc/gitconfig.

($ git add -h) Git-репозитория Репозиторий - это хранилище истории и данных вашего проекта. $ git init - создает новый репозиторий. $ git add - добавляет изменение из рабочего каталога в раздел проиндексированных файлов. Она сообщает Git, что вы хотите включить изменения в конкретном файле в следующий коммит. $ git commit -m - это команда для записи индексированных изменений в репозиторий Git. $ git clone - это процесс создания идентичной копии удаленного репозитория Git на локальную машину. Состояние файлов Помните, что каждый файл в вашем рабочем каталоге может находиться в одном из двух состояний: отслеживаемые или не неотслеживаемые (untracked). Отслеживаемые файлы — это те файлы, которые были в последнем коммите, они могут быть unmodified (немодифицированными), модифицированными (modified) или индексированный (staged). Если кратко, то отслеживаемые файлы — это те файлы, о которых знает Git. Неотслеживаемые файлы — это всё остальное, любые файлы в вашем рабочем каталоге, которые не входили в ваш последний коммит и не подготовлены к коммиту. $ git status - oпределение состояния файлов. $ git status -s, git status --short - cокращенное состояния файлов. $ git diff - чтобы посмотреть, что мы изменили, но пока не проиндексировали, показывает добавленные или удалённые строки кода. $ git diff --staged, git diff --cached - чтобы посмотреть, что мы проиндексировали и что войдёт в следующий коммит. Удаление файлов $ rm -> $ git rm - для того чтобы удалить файл из Git, необходимо удалить его из отслеживаемых файлов, а затем выполнить коммит, также удаляет файл из вашего рабочего каталога. Если файл не отслеживаемый (Untracked) то его можно удалить одной командой $ rm . $ git rm -f - удаление файлов в любом состояние, кроме не отслеживаемых (Untracked), также удаляет файл из вашего рабочего каталога. $ git rm --cached - удалить файл из индекса, при этом оставляет его в рабочем каталоге. $ git mv - переименовние файл. $ rm -rf .git - удаление локального репозитория. Просмотр истории $ git log - просмотр истории коммитов. $ git -p -2, git --patch -2 - показывает патч для каждого коммита, также можно ограничить количество отображаемых записей журнала, например, используя аргумент -2 для отображения только двух последних записей. $ git log --stat - показывает сокращенную статистику измененных файлов для каждого коммита. $ git log --pretty=oneline - выведет на каждый из коммитов по строчке, состоящей из хэша и название коммита. $ git log --pretty=short - выведит строке хэш, имя автора и название коммита. $ git log --pretty=full — более полная информация, выведит хэш, имя автора, имя коммитера и название коммита. $ git log --pretty=fuller - полная информация, выведит хэш, имя автора, дату, имя коммитера и название коммита. $ git log --pretty=format:'%()' - позволяет указать собственный формат вывода журнала. %H - хеш коммита. %h - cокращенный хеш коммита. %an - имя автора. %ae - электронная почта автора. %cn - имя коммитера. %ce - электронная почта коммитера. %s - название коммита. %ad - дата коммита автора. %ar - относительная дата коммита автора. %cd - дата коммита коммитера. %cr - относительная дата коммита коммитера. %T - хеш дерева. %t - сокращенный хеш дерева. %P - хеш родителей. %p - сокращенный хеш родителей. $ git log --graph - отображает ASCII граф с ветвлениями и историей слияний. $ git log --shortstat - oтображает только строку с количеством изменений/вставок/удалений для команды --stat. $ git log --name-only - показывает список измененных файлов. $ git log --name-status - показывает список файлов, которые были добавлены/изменены/удалены. $ git log --abbrev-commit - показывает только несколько символов хэш коммита вместо всех 40. $ git log --relative-date - отображает дату в относительном формате вместо стандартного формата даты. $ git log --oneline - покажет все коммиты только с первой частью хэша и названием коммита. $ git log -(n) - показывает только последние n коммитов. $ git log --since, git log --after - показывает только те коммиты, которые были сделаны после указанной даты. $ git log --until, git log --before - показывает только те коммиты, которые были сделаны до указанной даты. $ git log --author - показывает только те коммиты, которые сделанные определенным автором. $ git log --committer - показывает только те коммиты, которые сделанные определенным коммитером. $ git log --grep - показывает только те коммиты, которые содержат указанную строку, в название коммита. $ git log -S - показывает только те коммиты, в которых изменение в коде повлекло за собой добавление или удаление этой строки. $ git log -- - чтобы просмотреть историю, связанную с определеным файлом. $ git shortlog - с ее помощью можно сгруппировать коммиты по автору и вывести первые строки всех комментариев к ним. $ git log --no-merges - исключает из историю все слияний. $ git log --merges - показывает историю всех слияний. Операции отмены $ git commit --amend - это удобный способ изменить последний коммит. Очень важно понимать, что когда вы вносите правки в последний коммит, вы не столько исправляете его, сколько заменяете новым, который полностью его перезаписывает. В результате всё выглядит так, будто первоначальный коммит никогда не существовал, а так же он больше не появится в истории вашего репозитория. $ git restore --staged - для отмены индексации файла. $ git restore - oтмена изменений в файлe. Важно понимать, что git restore  — опасная команда. Любые локальные изменения, внесенные в этот файл, исчезнут — Git просто заменит файл последней зафиксированной версией. Tеги $ git tag - просмотр список имеющихся тегов. Данная команда перечисляет теги в алфавитном порядке, порядок их отображения не имеет существенного значения. $ git tag -l, --list * - показывает все теги начало которых соответствуют вашему фрагменту. $ git tag -l, --list ** - показывает все теги которые имеют сходство с вашем фрагментом. $ git tag -n - показывает имя тега и комминтарие к нему. $ git show - посмотр данных тега. $ git tag -a -m "commite-tag" - создание аннотированного тега, которые содержат в себе более подробную информацию, такую как имя создателя тега, адрес электронной почты, дата и коммит к тегу. $ git tag - создание легковесного тега, просто указатель на определённый коммит. $ git tag -a -m "commite-tag" , git tag - тег создается для того коммита на который ссылается указатель HEAD, но также можно указать тег (аннотированный, легковесный) на любой коммит в любое время. $ git push - по умолчанию, команда git push не отправляет теги на удалённый сервер. После создания теги необходимо явно отправить на удалённый сервер. $ git push --tags - отправляет все теги за один раз. $ git push --follow-tags - отправяляет только аннотированные теги. $ git tag -d - удаления тега в локальном репозитории. $ git push --delete - удаления тега в удаленом репозитории. $ git checkout - позваляет переключаться между различными версиями с помощью тегов. Псевдонимы $ git config --global alias. 'command' - создание alias. $ git config --get-regexp alias - показывает все имеющиеся alias, и их команды. Regexp = regular expression (регулярные выражение). $ git config --global --unset alias. - удаление alias. $ git config --global --edit - вызов консольного текстового редактора. Ветвление в Git $ git branch - создать новую ветку. $ git checkout - для переключения ветки. $ git log - для просмотра истории коммитов другой ветки. $ git checkout -b - создании новой ветки и сразу переход на неё.  $ git branch -v - чтобы посмотреть последний коммит на каждой из веток. $ git branch --merged - чтобы посмотреть те ветки, которые вы уже слили с текущей веткой. $ git branch --no-merged - чтобы увидеть все ветки, содержащие наработки, которые пока ещё не слили в текущую ветку. $ git branch --move - переименование ветки в локально репозитории. $ git push --set-upstream - переименование ветки в удаленом репозитории. $ git push --delete - удаление удаленой ветки. $ git branch -d - удаления ветки. $ git merge --no-ff - слияние веток без Fast-forward. $ git merge - слияние веток. $ git branch --all - список всех имеющихся локальных и удаленных веток. $ git branch -список всех локальных веток. Работа с удалёнными репозиториями $ git clone - это процесс создания идентичной копии удаленного репозитория Git на локальную машину. $ git remote - проверить с каким удаленным репозиторием git связывает локальный репозиторий. $ git remote -v - посмотреть URL удаленого репозитория. $ git remote add - добавить удалённый репозиторий. $ git remote remove(или rm) - убрать привязку удалённого репозитория. $ git remote rename - для переименования удалённого репозитория. $ git remote show - выводит информацию об этом репозитории. $ git fetch - получение изменений из удаленного репозитория. $ git fetch - данные извлекаются только для указанной ветки. $ git pull - получает изменения из переданного удаленного репозитория и обновляет рабочую копию в соответствии с удаленным репозиторием. $ git push - отправка изменений в удаленный репозиторий. $ git checkout -b , git checkout --track - создает локальную ветку на основе удаленной ветки, над которой можно работать. $ git push --delete - удалить ветку в удаленом репозитории (в локальном репо ветка остается не удаленой). $ git branch -vv - выведет список локальных веток и дополнительную информацию о том, какая из веток отслеживается, отстаёт, опережает. $ git branch -u - настраивает локальную ветку на слежение за удаленной веткой. $ git ls-remote - для получения полного списока удалённых ссылок. Удалённые ссылки — это ссылки (указатели) в удалённых репозиториях, включая ветки, теги и так далее. $ git push - запушить ветку в указанный репозиторий. $ git log --oneline --decorate --graph --all - для отображения истории коммитов, текущего положения указателей веток и истории ветвления. Перебазирование $ git rebase - переносит коммиты текущей ветки на вершину переданной.
